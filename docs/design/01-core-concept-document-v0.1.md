## 1. 愿景与调性

### 一句话游戏介绍

一款类 DNF 的 2.5D 战斗打怪与角色养成游戏：玩家可以手动刷图、连招、打 Boss，也可以让现实中的 AI 工具完成真实任务，将任务成果转化为角色成长、技能、装备、虚拟道具和副本进度。

### 双层调性

表层调性：

- 明亮、积极、热血、成长向
- 强调战斗爽感、技能反馈、副本推进、角色养成
- 玩家第一眼看到的是一个可以玩的动作刷图游戏

深层调性：

- 现代聊斋式 AI 寓言
- AI 不直接作为技术名词出现，而被抽象为“外智”“寄宿之力”“代役之灵”
- 暗线讨论 AI 时代的智力平权、智力贬值、成长断代、责任转移、阶层重塑

### 艺术方向关键词

- 2.5D 横版 / 斜视角战斗
- 东方幻想 + 现代技术隐喻
- 表面明亮、秩序、成长；深处有影、债、反噬与社会裂缝
- 主角可偏狂战士、刀剑、血气、重斩、残血反打
- 怪物与 Boss 可承载“失去本心的人”“被外智掏空的人”“被流程支配的空壳”等隐喻

## 2. 目标受众画像

### 核心受众 A：动作刷图玩家

- 喜欢 DNF、横版动作、技能连招、打击反馈、装备成长
- 关注手感、角色动作、怪物受击、Boss 压迫感
- 对 AI 暗线不一定敏感，但需要先被游戏本体吸引

### 核心受众 B：AI 工具重度使用者

- 使用 ChatGPT、Claude Code、Codex、VS Code 插件、MCP、自动化工作流
- 能理解“AI 替我干活，但我还要验收和背锅”的时代处境
- 对真实任务转化为游戏收益有兴趣

### 核心受众 C：开发者与创作者

- 关注 GitHub、CI、代码修复、文档生成、工具链、插件生态
- 愿意把真实产出映射到角色成长中
- 可能成为早期社区贡献者和工具接入者

### 潜在受众 D：喜欢世界观与社会寓言的玩家

- 喜欢表面游戏、深层隐喻的作品
- 关注 AI 时代对职业、阶层、劳动价值和个人成长的影响

## 3. 核心玩法循环

### 分钟级循环：手动战斗

```text
进入房间 / 副本
  ↓
移动、闪避、普通攻击、技能释放
  ↓
击败小怪 / 精英 / Boss
  ↓
获得掉落、经验、资源
  ↓
强化角色、技能、装备
  ↓
挑战下一波敌人或更高难度房间
```

### 小时级循环：成长与副本推进

```text
完成一组副本目标
  ↓
获得装备、技能点、资源
  ↓
构建角色流派
  ↓
解锁更高难度区域 / Boss
  ↓
验证手感、Build 和成长效果
```

### 中长期循环：外智代役与现实任务回流

```text
玩家选择现实任务
  ↓
调用 AI 工具 / 插件 / MCP / Skill / GitHub 流程
  ↓
AI 代役完成任务
  ↓
玩家审核与确认
  ↓
系统结算返还资源
  ↓
角色获得成长、道具、货币、技能强化或副本进度
  ↓
同时积累成本、风险、冷却、灵债或责任压力
```

## 4. 独特卖点

### 卖点 1：类 DNF 2.5D 战斗载体

项目不是纯 AI Dashboard，而是首先做成一个可玩的动作刷图游戏。玩家可以手动操作、打怪、放技能、打 Boss、养成角色。

### 卖点 2：真实任务成果转化为游戏成长

现实中的 AI 任务、代码修复、文档生成、CI 通过、PR 合并等行为，可以被转化为角色成长资源。游戏不只是模拟打怪，现实劳动会成为角色成长的一部分。

### 卖点 3：AI 时代的明暗双线世界观

表面是人人借助外智成长的含光盛世，背后是智力贬值、成长断代、责任转移和人被外智塑形的时代寓言。

### 卖点 4：AI 工具生态可扩展

后续可接入 VS Code、Claude Code、Codex、MCP、Skill、GitHub Actions、浏览器插件、Web 平台、App 打包等外部能力。

### 卖点 5：人机关系变成游戏机制

不是简单“AI 帮你挂机”，而是把外智使用的收益、代价、债务、审核、反噬、封印、本心等概念机制化。

## 5. 玩家动机与体验

### 成就型玩家

- 追求通关、副本挑战、Boss 击杀、角色成长、装备提升
- 现实任务完成也可转化为成就

### 探索型玩家

- 探索世界观暗线、人群分层、外智社会结构
- 发现表层积极世界背后的 AI 时代阴影

### 社交型玩家

- 后续可围绕任务大厅、贡献榜、角色展示、工具接入成果形成社区
- 玩家之间可以比较真实任务产出与角色成长路径

### 破坏 / 支配型玩家

- 挑战高难 Boss、极限连招、越级副本
- 后续可加入灰灯客、斩灯人等路线，探索规则边界

## 8. 平台与技术要求

### 当前首发平台

- Web / PC 浏览器
- 键盘操作优先
- 鼠标用于 UI、任务系统、Web 平台交互

### 当前技术栈

- Phaser 3
- TypeScript
- Vite
- Node >= 20
- GitHub 作为源码真源头
- GitHub Actions 作为后续真实验收环境

### 后续可能平台

- Web 平台完整化
- GitHub Pages / 静态部署预览
- 桌面 App 打包
- VS Code / 变种 IDE 插件生态
- MCP / Skill / Claude Code / Codex 工具接入

## 9. 内容量与生命周期：初步规划

### 原型阶段

- 1 名主角
- 3 类小怪
- 1 个 Boss
- 基础普通攻击、上挑、闪避、技能
- 1 个测试副本或房间
- 基础素材、动画、判定与打击反馈

### Demo 阶段

- 完整一关副本流程
- 至少 1 个可打 Boss
- 简单成长系统
- 初步外智 / 代役概念入口
- GitHub 任务或 CI 产物转化为游戏奖励的最小闭环

### 长期阶段

- Web 任务大厅
- 外智工具接入
- 多角色 / 多职业 / 多流派
- 虚拟收益、道具、称号、排行榜
- 赛季、活动、副本扩展

## 10. 概念层风险评估

| 风险 | 说明 | 应对 |
|---|---|---|
| 游戏本体不好玩 | 战斗手感不稳会导致所有深层设定失去载体 | 先把 2.5D 战斗 Demo 做扎实 |
| 世界观过阴暗 | 表面如果太压抑，不利于传播和玩家进入 | 表层保持积极、明亮、成长，暗线逐渐展开 |
| AI 系统太复杂 | 真实任务、工具接入、收益结算落地难 | 先从 GitHub / CI / 文档任务做最小闭环 |
| 命名不稳定 | 目前正式项目名未定 | 先使用暂定名，不影响内部文档推进 |
| 商业化与奖励风险 | 若涉及真实 CDK、月卡、虚拟币，需要防刷与合规 | 早期只做虚拟道具和测试奖励，不急于现金化 |
| 用户理解门槛高 | 明暗双线、外智隐喻可能过复杂 | 用可玩战斗承载，文案分层表达 |
